# goal.md (Owner-owned)

## 1) Intent / 原始意图
把多段、跳跃、重复的口述内容稳定转成可直接使用的文本，替代键盘长文本输入。

## 2) Target Outcome / 目标交付物
交付一个本地优先的 MVP：
- 多段语音会话（创建、录制、片段列表、删除、重试、重录）
- 分段转写与状态管理
- 合并转写并按模式整理输出
- 最终文本预览、复制、历史会话查看/再整理
- 代码与文档落盘在仓库内（`specs/` `plans/` `reports/` 与应用源码目录）

## 3) Success Criteria / 成功标准（可验证）
1. 用户可在一个会话中录制多个片段，片段可独立查看状态与转写结果。
2. 片段转写失败不影响其他片段，且可单独重试。
3. 点击完成整理后，可生成 `combined_transcript` 与 `final_text`。
4. 支持四种整理模式：原意清理、Obsidian 笔记、任务/需求整理、忠实转录。
5. 可复制最终文本并可从历史记录重新整理。
6. 架构约束满足：STT/Rewrite 均通过 adapter，Prompt 模板独立存放。

## 4) Scope / 范围
### In Scope
- 桌面端 MVP 闭环（多段录音 -> 分段转写 -> 合并整理 -> 输出）
- 本地存储（历史会话、片段元数据、文本结果）
- 统一错误处理与可见错误提示

### Out of Scope
- 单段录音作为独立主流程
- 完整智能体、语音操控电脑、实时字幕、多人会议纪要
- 移动端、账号系统、云同步、复杂知识库、自动调用外部开发流程

## 5) Constraints / 约束
- 本地优先；默认离线保存音频与会话元数据
- VoiceSession 与 VoiceSegment 必须分离建模
- Prompt 模板不得硬编码在主流程
- 禁止未经确认的破坏性操作
- 输出语言中文优先，风格简洁可执行

## 6) Inputs / 输入材料与上下文
- `project_control/.ggs/idea.md`（Owner 已填写主要考虑）
- 现有仓库当前为 GAEH 骨架，无业务代码

## 7) UI / Interaction Requirements（如涉及 UI）
核心交互：
创建会话 -> 录制第 N 段 -> 查看分段状态/文本 -> 删除或重试 -> 完成整理 -> 预览/复制 -> 历史查看与再整理。

第一阶段最小界面：
- 当前会话工作区
- 历史会话列表与详情
- 模式选择与整理触发

## 8) Boundary & Edge Cases / 边界与模糊点（重要）
- 空会话或全部片段失败时，禁止进入整理并给出明确提示。
- 删除片段后，合并顺序按现有有效片段 `order_index` 重新计算。
- 忠实转录模式不得重组原文结构；原意清理模式不得新增未表达观点。
- Provider 不可用、超时、返回空结果时，必须可见报错且可重试。

## 9) Output Format / 输出格式
- 工程产出：源码、配置、Prompt 模板、数据库 schema 与使用说明。
- 过程产出：`specs/` `plans/` `reports/` `reviews/` 与 `project_control` 状态文件持续同步。
- 可验证入口：本地运行命令 + 最小手工验收路径。

## 10) Risks / 风险与未知
- 桌面框架与本地服务选型仍存在取舍（Electron/Tauri，服务进程管理复杂度不同）。
- STT/LLM provider 的延迟与失败率会影响交互体验，需要重试与降级策略。
- 音频采集在不同设备上的稳定性（采样率/权限）存在环境差异风险。

## 11) Approval Policy / 同意门禁（Owner 决策）
Owner 已在对话中给出 `APPROVE`，允许按当前目标进入连续实施。
