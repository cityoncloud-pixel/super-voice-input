# mvp-0001.plan.md

## Plan Goal
按 `specs/mvp-0001.spec.md` 建立 MVP 基础骨架并打通最小闭环。

## Work Breakdown
1. **Project bootstrap**
   - 初始化应用目录（桌面端、本地服务、共享 domain）。
   - 建立基础配置与启动入口。
2. **Domain and storage**
   - 定义 VoiceSession/VoiceSegment 模型与状态枚举。
   - 建立本地 SQLite schema 与 repository 接口。
3. **Core pipeline**
   - 实现分段转写流程（含重试）。
   - 实现 transcript 合并与 mode rewrite 流程。
4. **Adapters and templates**
   - 建立 STT/Rewrite adapter 抽象与 mock provider。
   - 建立四种模式模板文件并接入 rewrite 调用。
5. **UI skeleton**
   - 会话工作区：录音列表、状态、操作按钮。
   - 结果区：模式选择、完成整理、预览、复制。
   - 历史区：列表与再整理入口。
6. **Verification**
   - 最小手工测试脚本与验收步骤。
   - 记录报告与后续风险。

## Execution Order
`bootstrap -> domain/storage -> adapters -> pipeline -> UI -> verify`

## Deliverables
- `specs/mvp-0001.spec.md`
- 基础源码目录与可运行入口
- `reports/mvp-0001.report.md`
- `project_control` 状态文件同步记录

## Risks and Mitigations
- 音频采集差异：先提供 mock 录音输入，真实录音分步接入。
- Provider 不稳定：默认提供 mock adapter，保证主流程可回归验证。
