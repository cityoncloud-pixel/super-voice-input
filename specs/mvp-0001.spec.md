# mvp-0001.spec.md

## Overview
本规格定义超级语音输入器 MVP 第一轮实现范围：从多段录音会话到最终整理文本输出的最小可运行闭环。

## Functional Scope
1. 创建会话并选择整理模式。
2. 在同一会话内录制多个片段。
3. 每个片段独立保存音频与状态。
4. 每个片段独立触发转写与重试。
5. 合并有效片段转写为 `combined_transcript`。
6. 调用重写适配器生成 `final_text`。
7. 展示预览、支持复制、保存历史并可再整理。

## Data Model
### VoiceSession
- id
- title
- mode
- status
- combined_transcript
- final_text
- rewrite_provider

### VoiceSegment

- duration_seconds
- raw_transcript
- stt_provider
- status
- created_at
- error_message

## Adapter Contracts
### STT Adapter
- `transcribe(audio_file_path: str, provider: str) -> raw_transcript`
- 失败时返回结构化错误，不中断其他 segment 处理。

### Rewrite Adapter
- `rewrite(mode: str, combined_transcript: str, provider: str) -> final_text`
- Prompt 模板从独立文件读取，不允许硬编码主流程。

## Modes
- `intent_cleanup`
- `obsidian_note`
- `task_requirement`
- `faithful_transcript`

## Error Handling
- 所有外部调用失败必须落库并可见提示。
- 会话级操作不得因为单 segment 失败而整体崩溃。
- 空输入、全失败、provider 不可用均要有明确错误消息。

## Non-Goals
- 单段录音独立主流程
- 实时字幕
- 云同步/账号系统
- 自动调用外部开发工具链

## Minimal Verification
1. 新建会话，录制 2 段，至少 1 段转写成功。
2. 删除 1 段后仍可成功完成整理。
3. 切换 4 种模式均可返回 `final_text`（可先用 mock provider）。
